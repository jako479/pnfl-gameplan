# Changelog

- `PnflGamePlan` wraps `GamePlan` + `PnflRules` + `PlayPool`.
- `PnflGamePlan` exposes `from_file`, `from_bytes`, `validate`, `save`.
- Add `PnflRules` + `OffenseCategoryRule` + `DefenseCategoryRule`.
- Add canonical `PNFL_RULES` instance.
- `validate_gameplan` returns `tuple[Violation, ...]`.
- Add `Violation` and `RuleName` (StrEnum) for structured violation reporting.
- Add `PnflRuleWarning(UserWarning)`.
- `PnflGamePlan.save()` is warn-and-persist (was raise-on-violation).
- `save()` emits one `PnflRuleWarning` per violation, prefixed `[pool_category]` when present.
- `save()` writes the file regardless of violations.
- `save()` returns `tuple[Violation, ...]`.
- `save()` logs an INFO summary line when violations are present.
- Switched per-violation reporting from `logger.warning` to `warnings.warn(..., PnflRuleWarning)`.
- Library no longer installs a `warnings` filter; consumers call `simplefilter("always", PnflRuleWarning)` at entry.
- Breaking change for consumers capturing violations via `caplog` only.
- Add `PnflGamePlan` property forwarders: `normal_plays`, `special_plays`, `clock_plays`, `custom_special_plays`, `stock_special_plays`, `is_offense`, `is_defense`.
- Add `PnflGamePlan.with_normal_plays` and `with_custom_special_plays` returning `PnflGamePlan`.
- Add `PnflGamePlan` `ClassVar` slot counts: `NUMBER_NORMAL_PLAYS`, `NUMBER_SPECIAL_SLOTS`, `NUMBER_SPECIAL_CATEGORIES`, `NUMBER_CLOCK_SLOTS`.
- Re-export `fbpro98-gameplan` value types (`CustomPlay`, `StockPlay`, `Play`, `ProfileType`) from `pnfl_gameplan`.
- Re-export `fbpro98-gameplan` I/O (`read_gameplan`, `parse_gameplan`, `write_gameplan`, `InvalidGamePlanError`) from `pnfl_gameplan`.
- Remove `PnflRuleError`.
- Remove `RuleName.CATEGORY_MIN_TWO_DL` (misread of per-play personnel rule).
- Remove `DefenseCategoryRule.min_two_dl` field.
- PassDazzle 50% 2-DL cap unchanged.
